Ping Service

A lightweight health-check microservice for monitoring uptime and connectivity across your infrastructure.

Overview

The Ping service is a minimal, zero-dependency HTTP server built with Node.js native modules. It provides a single /ping endpoint that returns pong - perfect for:

  • Health checks in container orchestration (Kubernetes, Docker Swarm)
  • Load balancer health probes (AWS ALB, Nginx, HAProxy)
  • Uptime monitoring (Datadog, Pingdom, UptimeRobot)
  • Service mesh connectivity verification
  • CI/CD pipeline smoke tests
Tech Stack: TypeScript, Node.js (native http module), tsx runtime

API Reference

GET /ping

Returns a plain text response confirming the service is alive.

Response
HTTP/1.1 200 OK
Content-Type: text/plain

pong

$ curl -i http://localhost:3000/ping

Source Code

The entire service is a single TypeScript file - simple, auditable, and easy to understand.

import http from "node:http";

const port = process.env.PORT ?? 3000;

const server = http.createServer((req, res) => {
  if (req.method === "GET" && req.url === "/ping") {
    res.writeHead(200, { "Content-Type": "text/plain" });
    res.end("pong");
  } else {
    res.writeHead(404, { "Content-Type": "text/plain" });
    res.end("Not Found");
  }
});

server.listen(port, () => {
  console.log(`Server listening on port ${port}`);
});
